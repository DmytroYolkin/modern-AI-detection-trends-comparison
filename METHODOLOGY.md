# Methodology

This document is a publication-ready description of the human-vs-AI text-detection pipeline implemented in this repository. It is what you would paste (with minor formatting) into the *Method* section of a paper.

Every claim below is verifiable from the code: file paths are linked, hyperparameters live in [`training/config.py`](training/config.py), and the dataset filters are recorded in [`data/features/meta.json`](data/features/meta.json).

---

## 1. Task and dataset

We address binary classification of natural-language passages into **human-written** vs. **AI-generated**, with a deliberate focus on the hard subcase where the AI text was produced by an LLM rewriting an existing human text (so topic, content, and semantics are matched between classes, and the only remaining signal is style).

### 1.1 Source corpora

The raw corpus combines three pools, harmonised into a single JSONL schema in [`data/dataset_ready_final/`](data/dataset_ready_final/):

| Pool | Label | Count | Description |
|---|---|---|---|
| USE student essays | `human` | 1,498 | Each essay carries a stable `author_id` (the human student). |
| LLM rewrites of those essays | `ai` | 6,051 | Five LLMs (LLaMA-3, Mistral, Gemma, Phi-3, Qwen2) paraphrased each essay; rewrites inherit their source essay's `author_id`. |
| HC3 / ArguGPT / RAID | mixed | 5,537 | Auxiliary AI-original and HC3-human data. Used only at intermediate stages; excluded from the final filtered training set (see §1.3). |

Each record schema: `id, text, label, source, author_id, model, source_text_id, split`.

### 1.2 Splitting

The split is **author-disjoint for USE** and **prompt-disjoint for HC3** (the latter does not matter under the §1.3 filter because HC3 is dropped). Seed `42`. Author-disjoint means every record by one student lands in exactly one split — so the TRACE author-fingerprint cannot leak across splits.

### 1.3 Two stacked sample-level filters

To make the experimental setup fully consistent with the TRACE framework (which assumes the existence of multiple human reference texts per author), two filters are applied during feature extraction:

- **`--require-known-author`** drops records whose `author_id` is unknown (`has_known_author == False`). This removes HC3 human answers (no per-author identity).
- **`--min-human-siblings 2`** drops records whose author contributes fewer than two *other* human texts as available TRACE context, excluding the source human of any LLM rewrite (to block original↔rewrite leakage; see §3.3).

After both filters, the kept set is **USE-only** (USE originals + their LLM rewrites for authors with ≥3 USE essays). Resulting per-split counts:

| Split | Records | Human | AI | Human:AI |
|---|---|---|---|---|
| train | 4,051 | 703 | 3,348 | 1 : 4.76 |
| val | 667 | 116 | 551 | 1 : 4.75 |
| test | 829 | 144 | 685 | 1 : 4.76 |

The class imbalance is intrinsic to the construction (one human essay yields ~5 LLM rewrites). Training compensates with inverse-frequency class weighting; the strict-FPR evaluation regime (see §6) is per-class-rate and unaffected by counts.

## 2. Three feature extractors

For each text the pipeline produces a `(87 + 10 + 128) = 225`-dimensional feature vector, computed once and cached to [`data/features/*.npz`](data/features/).

### 2.1 NELA (87-dim)

Adapted from the NELA toolkit [(Horne et al., 2018)](https://arxiv.org/abs/1803.10124). Surface-linguistic features:

- 49 POS-tag rates and punctuation rates (`JJ`, `NN`, `,`, `.`, …)
- 4 readability indices (Flesch-Kincaid, SMOG, Coleman-Liau, LIX)
- 3 lexical-diversity measures (TTR, avg word-length, word count)
- 6 hedging / assertativity / factive lexicon counts (Hedge, Factive, Assertative, Implicative, Report-verb, Bias)
- 9 sentiment scores (VADER + SentiWords positive/neutral/negative)
- 11 moral-foundations counts
- 2 named-entity counts (locations, dates)

Implementation in [`extractors/nela_extractor.py`](extractors/nela_extractor.py). All 87 NELA values for every sample in the final cache are non-zero (no extraction failures).

### 2.2 StyleDecipher (10-dim)

Compares each text against five LLM-generated paraphrases of itself and computes the mean and standard deviation of five similarity metrics:

- 1-, 2-, 3-gram Jaccard overlap
- Normalised edit similarity (`1 - Levenshtein / max_len`)
- Embedding cosine similarity (sentence-transformers `all-mpnet-base-v2`)

Rewrites are generated by five locally-hosted Ollama models (LLaMA-3, Mistral, Gemma, Phi-3, Qwen2) — `--styledecipher ollama` mode. Coverage is 100 % across every split in the released cache.

Implementation in [`extractors/styledecipher_extractor.py`](extractors/styledecipher_extractor.py).

### 2.3 TRACE (128-dim)

Author-style fingerprint based on the TRACE framework [(Liang et al., 2024)](https://arxiv.org/abs/2402.16638). Each text's TRACE embedding is computed from the text itself plus its *author context* (see §3.3 for the context-selection rule).

The pipeline applies the following steps:

1. **Principal-sentence extraction**: TF-IDF score every sentence in the input texts; keep the top 5 per text. This concentrates the embedding on the most stylistically marked sentences.
2. **Sentence-transformer encoding**: feed the kept sentences through `AnnaWegmann/Style-Embedding` ([Wegmann, Schraagen, Nguyen, 2022](https://aclanthology.org/2022.repl4nlp-1.26/), RepL4NLP) — a RoBERTa-base model contrastively fine-tuned on Reddit conversations for *content-independent style similarity*. The model is loaded with explicit `max_length=512` to honour its 514 max-position-embeddings limit.
3. **Mean pooling** of token embeddings under the attention mask (768-dim).
4. **Projection head** to 128-dim: a 2-layer non-trained linear-ReLU-linear network with fixed seed=42, acting as a Johnson-Lindenstrauss dimensionality reducer.
5. **Author-level mean pooling**: average the per-sentence 128-dim vectors into a single embedding for the anchor sample.

Implementation in [`extractors/trace_user_profiler.py`](extractors/trace_user_profiler.py), wired through [`training/extractor_pipeline.py`](training/extractor_pipeline.py).

#### Why the Wegmann backbone

The original TRACE paper requires a contrastively-trained sentence encoder. Lacking the released checkpoint, this work substitutes Wegmann et al.'s open-source `AnnaWegmann/Style-Embedding`, which is itself a contrastively-trained style encoder (NT-Xent over Reddit posts, same-author = positive). This avoids the methodologically-weird configuration of training nothing while keeping a randomly initialised projection: only the 128-dim projection layer is random; the underlying 768-dim representation is genuinely style-aware.

## 3. Fusion architecture

Implemented in [`fusion/combination_all.py`](fusion/combination_all.py) (`MultiFeatureFusion`) and wrapped by [`training/model.py`](training/model.py) (`FusionClassifier`).

### 3.1 Per-modality projection

Each of the three feature blocks (NELA / StyleDecipher / TRACE) is first projected to a shared `hidden_dim=256`-vector through an independent `Linear → ReLU → LayerNorm`. This puts all three modalities into a common space before fusion.

### 3.2 Fusion strategies (four variants)

| Strategy | Operation |
|---|---|
| `concat` | concatenate the three 256-vectors → 768-d → `Linear(768 → 256)` → ReLU → LayerNorm |
| `mlp` | same as `concat` but the post-projection is an explicit MLP block |
| `attention` | self-attention over the 3-vector sequence (4 heads), mean-pool the output |
| `gating` | learned gate `g = σ(Linear([n; s; t]))`, output `g · n + (1−g) · (s + t)/2` |

A 2-layer MLP head with dropout `0.3` maps the fused representation to two-class logits. Trained with class-weighted cross-entropy.

### 3.3 Author context and the no-source-leakage rule

The TRACE context for any anchor sample is defined as: *all other texts by the same author with `label == "human"`*, excluding the source text of an LLM rewrite. Concretely, in [`training/rebuild_trace_author.py`](training/rebuild_trace_author.py):

```python
def trace_human_siblings(sample, ds):
    out = []
    for s in ds._by_author[sample.author_id]:
        if s.record_id == sample.record_id:                              continue
        if s.label != "human":                                             continue
        if sample.is_rewrite and s.record_id == sample.source_text_id:   continue
        out.append(s)
    return out
```

The `is_rewrite ∧ same_as_source_text_id` exclusion is the key step that prevents the trivial leakage where an LLM rewrite's TRACE embedding is dominated by the embedding of the very text it was paraphrased from. Combined with the `min_human_siblings=2` filter (§1.3), every kept anchor has ≥2 human reference texts and the embedding is genuinely author-aware (validated by the per-split sibling stats `avg=3.47/3.34/3.62`, zero-sibling-samples=0).

## 4. Modality dropout (the multimodal-balance regulariser)

Without explicit regularisation, the optimiser collapses onto NELA's 87 surface features because they alone solve the USE-vs-rewrite task at 99 % accuracy ([§3 in `models/analysis.ipynb` §3b](models/analysis.ipynb)). To force the fusion head to also extract signal from StyleDecipher and TRACE, training applies **asymmetric per-block dropout** at the input of `FusionClassifier`.

In each training-mode forward pass, each feature block independently has its values zeroed for the whole sample with the following probabilities:

| Block | p(drop) |
|---|---|
| NELA | 0.5 |
| StyleDecipher | 0.2 |
| TRACE | 0.2 |

Kept activations are rescaled by `1/(1−p)` (inverted-dropout convention) so the expected magnitude at evaluation time is unchanged. At evaluation, dropout is disabled and all three blocks are visible.

This is the standard **modality dropout** technique from multimodal learning ([Neverova et al., *"ModDrop"*, TPAMI 2016](https://ieeexplore.ieee.org/document/7373645); cited 1000+). Asymmetric rates are a deliberate design choice — symmetric rates do not break the NELA collapse on this task, because NELA's marginal contribution per sample is large even when it appears with reduced frequency. By dropping NELA 2.5× more often than the other two, the optimiser is forced to also extract signal from the other modalities.

Implementation in [`training/model.py`](training/model.py) (`FusionClassifier._apply_modality_dropout`). Configurable via CLI: `--modality-dropout-nela`, `--modality-dropout-style`, `--modality-dropout-trace`. The values are persisted in every saved checkpoint's `hparams` so reload reconstructs the identical architecture.

## 5. Classical baselines

For each cached feature triple (`nela`, `style`, `trace`), the 225-dim flat concatenation is fed to six classical backends (in [`training/classical.py`](training/classical.py)):

- `xgboost` (XGBClassifier, hist tree method)
- `random_forest` (sklearn)
- `logreg` (L2-regularised, C=1, sklearn)
- `svm` (RBF kernel, sklearn)
- `mlp` (sklearn `MLPClassifier(256, 128)`)
- `hist_gbm` (sklearn `HistGradientBoostingClassifier`)
- `gradient_boosting` (sklearn, plain)

All classical models use inverse-frequency `class_weight` (where supported) and operate on the same feature normaliser that the neural fusion track uses.

Classical models do not see modality dropout — they are fitted directly on the 225-dim vectors. They serve as a reference for "how much does this task need a fusion architecture at all?".

## 6. Evaluation

### 6.1 Default-threshold metrics

At argmax (0.5 threshold), we report accuracy and macro-F1. ROC-AUC is reported for threshold-free ranking quality.

### 6.2 Strict-FPR threshold

In deployment, a false positive (a human text flagged as AI) is often much more costly than a false negative. We therefore also evaluate at the **lowest threshold on the val split for which val-FPR ≤ 1 %**, and report the test-set TPR / precision / FPR at that threshold.

### 6.3 Per-modality permutation importance

The pipeline-level question — *how much does each extractor actually contribute?* — is answered by **block-wise permutation importance**, which works for every model regardless of architecture:

1. Predict on the un-permuted test set; record baseline macro-F1 and AUC.
2. For each block ∈ {NELA, StyleDecipher, TRACE}:
   - Shuffle that block across the 829 test samples (10 independent random permutations, seed = 42).
   - Re-predict and record the mean drop in macro-F1 and AUC.

ΔF1 > 0 means the block was helping the model; ΔF1 ≈ 0 means the model effectively ignores it. The full importance table is reproduced in §7 below.

Implementation in [`models/analysis.ipynb`](models/analysis.ipynb) §3b.

## 7. Results

### 7.1 Test-set classification metrics

After all filters and the multimodal regularisation, all 11 models reach macro-F1 ≥ 0.99 on the held-out test (per [`models/analysis.ipynb` §2](models/analysis.ipynb)). Linear / SVM models specifically:

- `classical_svm`: val F1 = 1.0000, test F1 ≈ 0.9958
- `classical_logreg`: val F1 = 0.9974, ROC-AUC = 0.999

Three of the four neural fusion variants hit val F1 = 1.0000 (early-stopped at epoch 5). The fact that the task is fully solvable on the kept USE-only subset is the central methodological observation — it is what motivates the modality-dropout regulariser.

### 7.2 Per-extractor permutation importance

Test-set ΔF1 (positive = block carries signal). Higher = the model relies on that block.

| Model | NELA ΔF1 | StyleDecipher ΔF1 | TRACE ΔF1 |
|---|---|---|---|
| neural_attention   | +0.3407 | +0.0555 | **+0.0263** |
| neural_mlp         | +0.2864 | +0.0615 | **+0.0263** |
| classical_xgboost  | +0.4934 | +0.0030 | +0.0015 |
| classical_svm      | +0.4853 | +0.0066 | **+0.0215** |

For the two neural models (which receive modality-dropout regularisation), NELA's ΔF1 share drops from ~92 % (no dropout) to **76–80 %**, while TRACE's share rises from ~0 (statistical noise) to **6–7 %**. StyleDecipher contributes 13–17 % on the neural track. This is the central quantitative result of the multimodal-balance changes.

For the classical baselines (no dropout), only the linear models pick up additional TRACE signal, because tree splits find tight surface NELA thresholds that swamp marginal style information.

### 7.3 Effect of the Wegmann backbone in isolation

Compared to the prior cache built with `sentence-transformers/all-mpnet-base-v2` (a semantic-similarity encoder, not author-style), the Wegmann-encoded TRACE features lift `classical_svm` ΔF1 from `−0.0004` (effectively zero) to **+0.0215** — a 50× change — without any modality-dropout regularisation. This isolates the contribution of the backbone choice from the regulariser.

| Item | Value |
|---|---|
| Random seed | 42 (Python random, NumPy, PyTorch) |
